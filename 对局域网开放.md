# 对局域网开放

本文档说明如何让股票交易系统在局域网内通过前端端口（5173）访问。

---

## 架构说明

```
┌─────────────────────────────────────────────────────┐
│                    WSL / Ubuntu                      │
│                                                      │
│  ┌──────────────┐      Vite Proxy      ┌──────────┐ │
│  │  Vue 3 + Vite │  ──────────────────> │ Spring   │ │
│  │  (port 5173)  │   /api -> 127.0.0.1:8080  │  Boot    │ │
│  │               │                      │ (port    │ │
│  │  host: 0.0.0.0│                      │  8080)   │ │
│  └──────────────┘                      └──────────┘ │
│         │                                            │
│         │ 局域网访问                                   │
│         ▼                                            │
│  http://192.168.x.x:5173                             │
└─────────────────────────────────────────────────────┘
```

**关键思路**：只暴露前端 5173 端口到局域网，后端 8080 端口仅监听本地（127.0.0.1），由 Vite dev server 的 proxy 功能将 `/api` 请求转发到后端。

## 修改的文件

### 1. `frontend/vite.config.js` — Vite 开发服务器配置

```js
import { defineConfig } from 'vite';
import vue from '@vitejs/plugin-vue';

export default defineConfig({
  plugins: [vue()],
  server: {
    host: '0.0.0.0',       // 监听所有网络接口，允许局域网访问
    port: 5173,
    proxy: {
      '/api': {             // 将以 /api 开头的请求转发到后端
        target: 'http://127.0.0.1:8080',
        changeOrigin: true, // 修改请求 Origin 为目标地址
      },
    },
  },
});
```

- `host: '0.0.0.0'` — 允许通过本机 IP 访问，如 `http://192.168.1.100:5173`
- `proxy./api` — 前端请求 `/api/stocks` 时，Vite 自动转发到 `http://127.0.0.1:8080/api/stocks`

### 2. `frontend/src/api/http.js` — axios 基础路径

```js
const http = axios.create({
  baseURL: import.meta.env.VITE_API_BASE || '',
  timeout: 10000,
});
```

- 默认 `baseURL` 为空字符串 `''`，即请求当前前端站点自身的路径
- 例如 `http://192.168.1.100:5173/api/stocks`，由 Vite proxy 转发到后端
- 可通过环境变量 `VITE_API_BASE` 覆盖（如部署到生产环境时）

### 3. `backend/src/main/resources/application.yml` — Spring Boot 配置

```yaml
server:
  port: 8080
  address: 0.0.0.0
```

- `address: 0.0.0.0` 确保后端监听所有网络接口（虽然通过 proxy 访问时不需要，但保留以兼容直连调试）

### 4. `backend/src/main/java/com/stocktrading/config/SecurityConfig.java` — CORS 配置

```java
configuration.addAllowedOriginPattern("*");
```

- 允许任意来源的跨域请求（Vite proxy 转发时 Origin 可能变化）
- 保留 `Authorization`、`Content-Type` 请求头
- 保留 `GET`、`POST`、`PUT`、`DELETE`、`OPTIONS` 方法
- Spring Security 权限控制保持不变

## 启动方式

**终端 1 — 启动后端：**
```bash
cd backend
mvn spring-boot:run
```

**终端 2 — 启动前端：**
```bash
cd frontend
npm run dev
```

## 验证方式

### 1. 获取本机局域网 IP

```bash
ip addr show | grep 'inet ' | grep -v 127.0.0.1
```

或

```bash
hostname -I
```

假设获取到 IP 为 `192.168.1.100`。

### 2. 验证步骤

| 步骤 | 地址 | 预期结果 |
|------|------|---------|
| 后端直连 | `http://localhost:8080/api/stocks` | 返回股票 JSON |
| Vite proxy | `http://localhost:5173/api/stocks` | 返回股票 JSON（Vite 转发到 8080） |
| 前端页面 | `http://localhost:5173` | Market 页面正常显示股票数据 |
| 局域网访问 | `http://192.168.1.100:5173` | 前端页面正常，API 通过 Vite proxy 工作 |
| 局域网 API | `http://192.168.1.100:5173/api/stocks` | 返回股票 JSON |

### 3. 常见问题

**Q: 局域网其他设备无法访问 `http://192.168.1.100:5173`？**

A: 检查防火墙是否放行 5173 端口：
```bash
# Ubuntu / WSL
sudo ufw allow 5173

# 或临时关闭防火墙测试
sudo ufw disable
```

**Q: 页面能打开但 API 请求失败？**

A: 检查后端是否已启动，以及 Vite proxy 配置是否正确。打开浏览器开发者工具 → Network 标签，查看请求是否发到 `localhost:5173/api/...` 并返回 200。

**Q: 登录后提示"请先登录"？**

A: 检查 `http.js` 中的 `baseURL` 是否为空字符串。如果设置了 `VITE_API_BASE=http://localhost:8080`，局域网其他设备访问时会尝试请求自己本机的 8080 端口，导致失败。注释掉或删除该环境变量即可。

## 生产环境部署

当前方案适用于开发环境（Vite dev server）。生产环境建议：

1. 构建前端静态文件：`cd frontend && npm run build`
2. 将 `dist/` 目录部署到 Nginx / Caddy
3. 在 Nginx 中配置反向代理 `/api` 到后端 `http://127.0.0.1:8080`
4. 后端 `application.yml` 中 `address` 可改回 `127.0.0.1` 以增加安全性
