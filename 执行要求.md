模块 4：股票中央交易系统

实现股票撮合交易逻辑。要求：

1. 用户提交买入或卖出委托后，系统先保存委托
2. 委托字段至少包括：
   - 委托 ID
   - 用户 ID
   - 股票代码
   - 买卖方向：BUY / SELL
   - 委托价格
   - 委托数量
   - 已成交数量
   - 剩余数量
   - 委托状态
   - 创建时间
3. 委托状态至少包括：
   - PENDING：等待撮合
   - PARTIAL_FILLED：部分成交
   - FILLED：全部成交
   - CANCELLED：已撤销
4. 撮合规则：
   - 只匹配同一只股票
   - 买单价格 >= 卖单价格时可以成交
   - 优先按照价格优先、时间优先原则撮合
   - 买单按照价格从高到低排序，价格相同按时间从早到晚
   - 卖单按照价格从低到高排序，价格相同按时间从早到晚
5. 成交价格可以使用卖方价格，或者使用先进入市场的委托价格。请在代码注释中说明你采用的规则。
6. 成交后需要：
   - 生成成交记录 Trade
   - 修改买方资金账户余额和冻结金额
   - 修改卖方证券账户持仓
   - 修改买方证券账户持仓
   - 修改卖方资金账户余额
   - 修改买卖双方委托的已成交数量、剩余数量和状态
7. 买入委托提交时，需要冻结预计资金：
   - 冻结金额 = 委托价格 × 委托数量
8. 卖出委托提交时，需要检查持仓数量是否足够，并冻结对应股票数量
9. 使用数据库事务保证撮合过程的数据一致性

9. 所有金额使用 BigDecimal，不要使用 double
