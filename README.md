# Stock Trading System

## Project Overview
This is a course-design level simulated stock trading system. It does not connect to real markets and uses mock trading logic with price-time priority matching.

## Tech Stack
- Backend: Java 17, Spring Boot 3, Maven, Spring Web, Spring Data JPA, Spring Security
- Frontend: Vue 3, Vite, Axios, Element Plus, Pinia
- Database: MySQL (local VM instance), optional H2 profile

## Project Structure
- backend: Spring Boot application
- frontend: Vue 3 application

## Prerequisites
- Java 17+
- Node.js 18+
- MySQL 8+ (local VM instance)

## Database Setup (Local VM MySQL)
MySQL is expected to run directly on the VM (no Docker).

Create database and user (run as root in MySQL):
```sql
CREATE DATABASE IF NOT EXISTS stock_trading CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;
CREATE USER IF NOT EXISTS 'stock_user'@'localhost' IDENTIFIED BY 'stock_pass';
CREATE USER IF NOT EXISTS 'stock_user'@'127.0.0.1' IDENTIFIED BY 'stock_pass';
GRANT ALL PRIVILEGES ON stock_trading.* TO 'stock_user'@'localhost';
GRANT ALL PRIVILEGES ON stock_trading.* TO 'stock_user'@'127.0.0.1';
FLUSH PRIVILEGES;
```

Default backend config in [backend/src/main/resources/application.yml](backend/src/main/resources/application.yml):
- URL: jdbc:mysql://127.0.0.1:3306/stock_trading
- User: stock_user
- Password: stock_pass

## Run Backend
```bash
cd backend
mvn spring-boot:run
```

## Run Frontend
```bash

```

## Core Features
- Auth with first-login certificate verification
- Fund account and securities account management
- Stock list and search (public)
- Buy/sell order submission with fund/holding freeze
- Matching engine (price-time priority; trade price = sell price)
- Order and trade history
- Admin dashboards for orders, trades, stats, users, accounts
- Staff dashboards for user and account operations

## Testing
### Backend smoke test
1. Start MySQL and backend.
2. Login as user1 (first-login with CERT-USER1).
3. Submit buy/sell orders and confirm matching creates trades.

### Frontend smoke test
1. Login as admin/staff/user1.
2. Verify pages: Market, Fund, Holdings, Trade, Orders, Trades.
3. Admin: check Orders/Trades/Stats/Accounts tabs.
4. Staff: create user, fund account, securities account, and link accounts.

## Notes
- Matching uses price-time priority and executes at sell price.
- Amounts use BigDecimal; DB tables are generated by JPA on first run.
