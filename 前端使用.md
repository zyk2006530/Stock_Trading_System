# 股票交易系统前端使用指南
1. 启动系统
先启动后端：
cd backendmvn spring-boot:run
再启动前端：
cd frontendnpm installnpm run dev
浏览器打开：
http://localhost:5173
项目 README 里也写明后端使用 Spring Boot，前端使用 Vue 3 + Vite + Axios + Element Plus + Pinia，前端启动命令是 npm run dev。

2. 初始账号
初始账号来自后端初始化文件：
backend/src/main/java/com/stocktrading/init/DataInitializer.java
其中创建了三个用户：admin、staff、user1，密码都是 123456；user1 是首次登录用户，证书码是 CERT-USER1。

3. 登录方式
打开前端页面后，右上角点击 Login。
登录页有两个标签：
LoginFirst Login
源码位置是：
frontend/src/views/LoginView.vue
其中 Login 标签只需要用户名和密码，First Login 标签需要用户名、密码和证书码。
普通登录
管理员和工作人员直接使用 Login 标签登录：
Username: adminPassword: 123456
或者：
Username: staffPassword: 123456
user1 首次登录
user1 第一次登录需要切到 First Login 标签：
Username: user1Password: 123456Certificate Code: CERT-USER1
首次登录成功后，以后 user1 就可以直接用 Login 标签：
Username: user1Password: 123456

5. 推荐演示流程
你可以按这个顺序完整使用一遍系统：
1. 打开 http://localhost:51732. 不登录，查看 Market 股票行情3. 点击 Login4. 切到 First Login5. 使用 user1 / 123456 / CERT-USER1 登录6. 进入 Fund，查看资金账户7. 进入 Holdings，查看持仓8. 进入 Trade，提交买入或卖出委托9. 进入 Orders，查看委托记录10. 进入 Trades，查看成交记录11. Logout12. 使用 staff / 123456 登录13. 进入 Staff Dashboard，查看工作人员账户管理功能14. Logout15. 使用 admin / 123456 登录16. 进入 Admin Dashboard，查看系统管理功能

8. 使用时最容易搞错的地方
staff 不是交易用户，所以不要用 staff 去点 Fund、Holdings、Trade。现在你的菜单已经按角色隐藏了这些入口。工作人员应该进 Staff Dashboard。
admin 也不是交易用户，管理员应该进 Admin Dashboard。
user1 才是普通股票交易用户。你要演示买卖股票、资金账户、持仓、委托、成交，应该用 user1 登录。
首次登录 user1 必须用 First Login 标签，填证书码 CERT-USER1。成功一次后，后续才可以用普通 Login 登录。
